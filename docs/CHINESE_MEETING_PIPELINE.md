# 中文会议提取 Pipeline 说明

本项目的新主线面向中文课堂/会议场景，不再把英文双人盲源分离模型作为唯一核心能力，而是采用：

```text
会议音频 -> DeepFilterNet增强 -> FunASR/SenseVoice中文ASR -> VAD + CAM++说话人分段
-> 说话人轨道 -> 主题转写 -> 会议纪要 -> 质量评分与可解释诊断
```

## 为什么优先说话人分段

真实会议多数时间是多人轮流说话，只有少量抢话和重叠。直接用盲源分离模型把整段会议硬拆成多条波形，容易受到语言、说话人音色相近、噪声和时长影响。对会议纪要系统来说，更关键的是知道“谁在什么时候说了什么”。

因此默认策略是：

- 用 FunASR/SenseVoice 负责中文转写。
- 用 `fsmn-vad` 定位有效语音片段。
- 用 `cam++` 生成说话人标签。
- 根据说话人时间段导出可试听轨道。
- 对短时高重叠片段保留 ClearVoice/MossFormer2 实验分离接口。

## 答辩创新点

- 中文会议自适应路由：根据模型可用性、说话人分段和疑似重叠比例选择处理路径。
- 说话人会议画像：统计每个说话人的发言时长、轮次、最长连续发言和参与占比。
- 会议提取质量评分：综合语音覆盖率、重叠比例、ASR状态和轨道生成状态输出 0-100 分。
- 可解释诊断：前端直接展示主处理后端、中文ASR模型、说话人分段模型和路由说明。

推荐答辩表述：

> 本项目针对中文会议场景，将传统“增强 + 英文盲源分离 + ASR”的拼接流程，优化为“增强 + 中文会议转写 + 说话人分段 + 自适应质量评估”的可解释流水线。系统不仅输出会议纪要，还能说明使用了什么模型、为什么选择该路径、每位说话人的参与情况以及本次提取质量。
